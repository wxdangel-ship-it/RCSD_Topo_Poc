# 2026-06-10 模块级文档契约审计

## 1. 范围

- 审计对象：`T08 / T01 / T07 / T03 / T04 / T05 / T06 / T09 / T10`
- 审计目标：确认模块级文档契约结构是否一致，以及当前内容是否足以承载凝练版需求说明与详细版需求说明。
- 非目标：不改写模块业务事实，不调整模块接口，不重命名入口，不替代逐模块需求治理。

## 2. 统一契约口径

| 文档 | 应承载内容 | 不应承载内容 |
|---|---|---|
| `README.md` | 模块阅读入口：模块一句话定位、当前状态、文档职责表、阅读顺序、入口位置。 | 凝练版需求正文、详细实现过程、长篇运行手册、完整参数表、历史过程记录。 |
| `SPEC.md` | 凝练版需求说明：模块定位、业务目标、范围 / 非目标、上下游、输入输出、关键业务步骤、对错边界和治理缺口。 | 运行教程、完整参数表、实现文件清单。 |
| `architecture/04-solution-strategy.md` | 详细版需求 / 落地策略：每个业务步骤的业务目的、输入前提、落地策略、输出审计和对错边界。 | 仅由伪代码、参数定义、实现文件名或入口命令组成的说明。 |
| `INTERFACE_CONTRACT.md` | 稳定契约：正式范围、输入、输出、业务规则、入口、参数类别和验收口径。 | 可迁移到 `README.md` 的阅读说明，或可迁移到 `architecture/04` 的详细步骤解释。 |
| `architecture/05-building-block-view.md` | 实现构件职责映射。 | 业务目标和接口契约正文。 |
| `architecture/10-quality-requirements.md` | CRS、拓扑、几何语义、审计、性能、验收质量要求。 | 运行教程和临时样例记录。 |
| `architecture/11-risks-and-technical-debt.md` | 当前风险、技术债与后续治理边界。 | 已完成事项的过程复述。 |
| `architecture/12-glossary.md` | 模块局部术语与字段名解释。 | 全局 SWSD / RCSD 术语重复说明。 |

## 3. 总体结论

1. 目标模块均具备 `README.md`、`AGENTS.md` 与 `INTERFACE_CONTRACT.md`，因此“标准模块文档面”已经成立。
2. “需求契约一致性”尚未成立：多个模块仍把凝练需求、运行说明、详细步骤和接口契约混写在同一文档中。
3. 经用户确认，后续凝练版模块业务需求应由模块级 `SPEC.md` 承载，`README.md` 回归模块阅读入口。
4. `T09` 最接近此前模板口径，但也需要迁移到 `README.md` 索引 + `SPEC.md` 凝练需求的新职责模型。
5. `T06` 是结构差异最大的模块，仍使用旧的 `02-business-rules / 03-input-output-contract / 04-algorithm-strategy` 架构命名。
6. 下一轮不宜一次性重写所有模块正文，应按模块逐个确认业务事实后，再把内容迁移到统一契约结构。

## 5. 治理优先级

| 优先级 | 模块 | 原因 |
|---|---|---|
| P0 | `T06` | 当前结构与统一契约偏差最大，且位于 T05 到 T09 的关键承接点。 |
| P1 | `T05`、`T08` | 文件结构缺口明显，且是全链关系融合与全局预处理核心模块。 |
| P1 | `T01`、`T03` | 业务内容充足，但详细需求与接口契约职责混杂，影响后续人机协作阅读。 |
| P2 | `T07`、`T10` | 文件结构基本齐全，主要需要补齐凝练版需求表达与详细步骤展开。 |
| P2 | `T04`、`T09` | 当前口径较稳，后续以小幅补齐和保持一致为主。 |

## 6. 建议治理路径

1. 先固定模块文档契约模板：`README.md` 为阅读入口，`SPEC.md` 为凝练版需求，`architecture/04` 为详细版需求 / 落地策略。
2. 按 `T08 -> T01 -> T07 -> T03 -> T04 -> T05 -> T06 -> T09 -> T10` 逐模块治理，不跨模块批量改写业务事实。
3. 每个模块先补 `SPEC.md` 的凝练版需求，再补 `architecture/04` 的详细版需求，最后把 `README.md` 收敛为索引。
4. 对 `T06` 先做结构迁移方案，再决定是否保留旧文件为兼容索引，避免破坏现有引用。
5. 每个模块完成后同步检查 `AGENTS.md` 阅读顺序、`module-doc-status.csv` 状态和项目级盘点中的模块缺口。
