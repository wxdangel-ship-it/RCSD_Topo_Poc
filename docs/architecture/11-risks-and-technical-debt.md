# 11 风险与技术债

## 当前主要风险

1. 当前输入数据组织方式只是兼容参考仓库，未来可能与 RCSD 真实业务需求分叉。
2. 当前没有业务模块，后续首次落地模块时仍需补模块级真实约束。
3. 共享协议当前是仓库级通用版本，后续某些模块可能需要扩展指标与断点枚举。

## 当前可接受技术债

- 当前 `tools/`、`scripts/` 只有最小占位，不提供专项业务入口。
- 当前没有项目级历史文档与变更档案，后续会逐轮积累。

## 后续缓解方式

- 任何 RCSD 模块启动任务都先基于模板补齐文档契约
- 数据布局调整必须通过独立任务写回项目级文档
- 共享协议扩展时同步补测试
