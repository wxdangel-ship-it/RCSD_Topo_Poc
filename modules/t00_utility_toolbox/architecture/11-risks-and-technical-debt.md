# 11 风险与技术债

## 当前主要风险

- T00 可能被误扩展为业务生产模块或“杂项脚本堆积区”
- Tool1 可能在实现时被顺手扩展为点云、轨迹或几何处理流水线
- Windows 路径与 WSL 路径若映射不清，容易导致后续实现时读写目标错误

## 当前可接受技术债

- 本轮只建立文档基线，尚未明确日志文件命名与摘要文件命名
- `history/*` 暂未建立，待进入真实迭代后再补
- 当前不为 Tool2+ 预建设计

## 后续缓解方式

- 进入编码前再次以 `spec.md` 与 `INTERFACE_CONTRACT.md` 做实现前复核
- 在 Tool1 编码任务书中明确脚本落点、日志命名和最小验证方式
- 若新增工具，逐个补规格并单独评审
