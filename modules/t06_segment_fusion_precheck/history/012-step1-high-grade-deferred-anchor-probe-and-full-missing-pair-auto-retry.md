# 012 Step1 高等级延迟锚定放行与两端缺失 pair 自动重试

- 时间：2026-06-13
- 背景：991176 Case 中 `991159_1049534`、`63010035_504467861`、`1020024_504467861`、`987957_988094` 均在当前正式结果中停于 T06 Step1，原因是部分 `has_evd=yes` 的 SWSD 语义节点仍为 `is_anchor=no`。这些 Segment 未进入 Step2，导致既有 buffer-only probe、pair anchor 自动补全、formal retry 和 failure business audit 均无机会运行。
- 根因：T07/T05 对 T 型/分歧合流等非 1V1 语义路口保守输出失败 relation，T06 Step1 又把 `is_anchor` 作为进入 Step2 的硬门槛，导致 T06 无法使用自身 relation-independent 的 RCSDRoad/RCSDNode 空间拓扑证据验证是否可构建 RCSDSegment。离线模拟用 Step1 candidates 直接喂 Step2 后，`991159_1049534`、`63010035_504467861`、`1020024_504467861` 已能通过正式 Step2 硬审计；`987957_988094` 虽有高置信 buffer-only corridor，但原 pair 两端 relation 均失败，旧逻辑只允许单端缺失或已有两端错误重配，因而仍被拒绝。
- 变更一：Step1 对 `sgrade` 属于 `0-0* / 0-1*` 的高等级 Segment 增加受限放行。仅当节点 `has_evd=yes`、`is_anchor` 明确为非可用 anchor 时，`pair_nodes.kind_2=2048` 与 `junc_nodes.kind_2 in {16,2048}` 可进入 `t06_swsd_segment_final_fusion_units`，交由 Step2 probe 与硬审计判定；该规则不改变 `is_anchor` 事实，不回写 T05 relation，不把这些节点写入 `junc_kind2_exempt_nodes`。
- 变更二：`pair_anchor_auto_retry` 增加“两端 pair relation 均缺失”的高置信自动补全分支。只有 `BufferOnlyProbe` 输出 `corridor_found`、`manual_review_required=False`、`repair_recommendation=high_confidence_pair_anchor_candidate`、connectivity/directionality 均为 `1.0`、shape similarity 不低于 `0.95`，且 pair anchor diagnostic 认为候选可自动修复时，才用候选 pair 构造当前 Segment 内的 effective relation；随后仍必须通过 Step2 buffer / direction / topology / geometry / leaf endpoint 硬审计，失败则保持 rejected。
- 变更三：对侧保持单端补全的 failure business audit，同步将 `invalid_pair_relation_status / invalid_pair_base_id` 与 `missing_pair_relation` 一样归一为自动补全审计口径；只要最终 Segment 已通过正式硬审计进入 replaceable，`manual_review_required` 不再保留为 `True`，`repair_recommendation` 统一记录为 `side_preserving_missing_pair_anchor_completion`。
- 变更四：Step2 在写出 `t06_rcsd_repair_candidates.*` 前，按最终 `t06_rcsd_segment_failure_business_audit.*` 的 replaceable 结论回填同 Segment 的自动补全口径。对已通过正式硬审计的 `high_confidence_pair_anchor_candidate` 与 `side_preserving_missing_pair_anchor_completion`，repair candidates 不再保留 buffer-only 原始 `manual_review_required=True`，避免人工审计看到中间诊断与最终业务结论冲突；`t06_rcsd_buffer_only_probe.*` 仍保留原始 probe 结果。
- 变更五：`pair_anchor_auto_retry` 对高等级 single 增加“单端缺失 + 已知端也被 high-confidence candidate 判定为错误锚点”的组合修复。旧逻辑只允许保留 T05 已知端点并补缺失侧，若已知端点本身不在 high-confidence candidate pair 中，则直接拒绝；新逻辑要求 `candidate_anchor_mismatch` 诊断明确覆盖两个 SWSD pair 端点、原始错误端包含一个已知 RCSD anchor 与一个缺失端、candidate pair 与 probe 首选一致，且仍满足非人工复核、连通/方向满分、shape similarity 不低于 `0.95`。通过后只在当前 Segment 内构造 effective relation，并继续执行 Step2 正式硬审计。
- 变更六：高等级 single graph-first 纵向重审增加 `rcsd_directed_path_missing` 来源。该路径仍只适用于原始 T05 pair relation 不变的 single Segment，不消费 pair anchor repair candidate；graph-first 必须在全 RCSDRoad 有向图中联通 pair 路口，经过 50m buffer core，并通过路径长度比例、首尾外延与 75m/100m 几何参考覆盖门槛。
- 安全边界：本轮不新增输入字段、不猜测 `kind_2` 新语义、不新增执行入口、不修改 T05 relation、不把 Step1 放行节点视作 anchor 成功。两端缺失 pair 的候选顺序只来自 relation-independent buffer-only probe 的 SWSD geometry endpoint 证据；若 probe 判定 ambiguous、人工复核、方向/连通/形态任一不满门槛，则不自动替换。
- GIS / 拓扑检查：991176 审计使用 `EPSG:3857` 统一计算距离与 buffer；四条目标 Segment 的 50m buffer、候选 RCSDRoad、候选 RCSDNode 与图连通结果已输出到 `outputs/_work/t10_991176_deep_audit_20260613/`。Step2 仍执行 RCSDRoad buffer overlap、canonical RCSD node 图连通、双向/单向通行方向、retained leaf endpoint、额外 mapped semantic node 与几何覆盖审计。
- 回归：新增 `test_step1_allows_high_grade_deferred_anchor_nodes_for_step2_probe`，覆盖高等级延迟锚定节点进入 Step2 probe、低等级同类节点仍被 Step1 拒绝；新增 `test_step2_autofills_fully_missing_pair_from_high_confidence_buffer_probe`，覆盖两端 pair relation 均失败但 buffer-only 高置信 corridor 经正式 Step2 硬审计进入 replaceable；新增 `test_step2_retries_missing_pair_with_known_anchor_mismatch`，覆盖单端缺失且已知端也被判错的 high-confidence candidate anchor mismatch；扩展 single graph-first 测试，覆盖 `rcsd_directed_path_missing` 作为纵向重审来源；扩展既有单端补全测试，覆盖 `invalid_pair_relation_status` 的审计归一，并断言 repair candidates 与 failure business audit 对自动补全口径一致。
