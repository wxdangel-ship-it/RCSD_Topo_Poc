# T00 模块规格：项目工具集合

## 1. 模块定位

T00 是项目工具集合模块，承载历史一次性预处理、格式转换、数据归位和辅助检查脚本。它不属于当前业务生产闭环，不直接产出 SWSD-RCSD 语义关系、Segment 替换结果或 F-RCSD 通行规则；它的价值是让历史辅助工具有统一位置、可追溯入口和最小治理边界。

## 2. 业务目标

- 将历史散落的数据整理、格式转换、字段补充和批量导出工具保留在统一模块中。
- 保证工具输入、输出、CRS、覆盖策略和失败原因可追溯。
- 为 T08 正式预处理模块迁移或吸收历史工具能力提供比较依据。
- 避免 T00 演化成新的业务生产模块或隐式主链入口。

## 3. 当前范围

### 3.1 支持保留

- Patch 目录骨架整理和 `Vector/` 数据归位。
- DriveZone / Intersection / DivStripZone 的历史逐 Patch 预处理与汇总。
- A200 Road / Node 的历史字段补充与导出。
- 顶层 GeoJSON 批量转 GPKG。
- JSON 上车点导出为 GPKG。
- MIF 转 GeoJSON / GPKG。

### 3.2 当前非目标

- 不构建 SWSD Segment。
- 不生成路口 1:1 relation。
- 不生成虚拟路口面。
- 不执行 T06 Segment 替换。
- 不还原 F-RCSD 通行规则。
- 不替代 T08 正式预处理模块。

## 4. 上下游关系

| 方向 | 模块 / 数据 | 关系 |
|---|---|---|
| 上游 | 原始 Patch / A200 / MIF / JSON 等历史数据 | 提供工具输入。 |
| 下游 | T08 | T08 可吸收、替代或复用 T00 中已经稳定的预处理经验。 |
| 支撑 | 人工排查 / 历史回归 | T00 输出可用于历史问题复现和辅助比较。 |

## 5. 输入

| 输入 | 用途 |
|---|---|
| Patch 目录 | 归位矢量数据并形成可复用目录骨架。 |
| DriveZone / Intersection / DivStripZone | 历史面数据预处理与汇总。 |
| A200 Road / Node | 历史字段补充和节点导出。 |
| GeoJSON / MIF / JSON | 格式转换或点位导出。 |

## 6. 输出

| 输出 | 用途 |
|---|---|
| `patch_all` 目录结构 | 历史数据整理结果。 |
| `DriveZone_fix / Intersection / DivStripZone` 汇总成果 | 供历史流程或人工对比使用。 |
| A200 Road/Node 派生成果 | 历史字段补充和数据检查依据。 |
| GPKG / GeoJSON 转换成果 | 统一格式后的辅助输入。 |
| summary / report | 记录输入、输出、CRS、计数、失败原因和覆盖行为。 |

## 7. 关键业务原则

- 每个工具都应回答“读了什么、改了什么、输出到哪里、失败了什么”。
- CRS 缺失或不明确时不得靠坐标值猜测业务语义。
- 已存在输出按工具契约覆盖或重建，不在输入文件上原地修改。
- T00 的结果是支撑材料，不自动成为 T01-T10 的正式 handoff。

## 8. 什么是对

- 工具行为轻量、可复跑、可审计。
- 输出保留足够输入路径、CRS、计数和异常信息。
- 与正式业务链有关的能力迁移到 T08 或对应业务模块后，T00 只保留历史追溯角色。

## 9. 什么是错

- 把 T00 工具输出直接当作当前主链正式成果。
- 在 T00 中新增新的业务算法或长期业务规则。
- 靠局部数据样本反推字段语义并固化到工具里。
- 新增工具入口但不走仓库入口治理。

## 10. 当前治理缺口

- 历史工具与 T08 正式预处理能力的边界需要持续保持清晰。
- 后续若 T08 已完全吸收某项工具能力，应在文档中明确 T00 仅保留历史入口。
