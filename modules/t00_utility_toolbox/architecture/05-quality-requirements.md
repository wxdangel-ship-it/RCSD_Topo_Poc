# 05 Quality Requirements

## 1. 业务正确性

- T00 必须保持支撑工具定位，不产出 relation、Segment 替换或通行规则。
- 工具只能执行已声明的数据整理、字段补充、格式转换或点位导出。
- CRS 缺失或不可信时不得靠坐标样本猜测业务语义。

## 2. 输出与契约稳定性

- 每个工具必须可复跑。
- 旧输出已存在时按契约删除重建，不能混用新旧结果。
- summary 必须足以说明输入、输出、计数、失败和跳过原因。
- Tool7 / Tool11 目录模式只扫描顶层文件，不递归扩张。

## 3. Review 与 formal 分层

- T00 当前无稳定 review-only 输出。
- 人工检查不能把 T00 输出提升为 T01-T10 正式 handoff。

## 4. 观测、恢复与性能

- 命令行执行必须输出阶段级进度。
- per-patch / per-file / per-record 工具必须记录处理进度。
- Tool11 GeoJSON / GPKG 写出优先采用 `ogr2ogr` 原生转换路径；fallback 写出引擎必须写入 summary。

## 5. 治理要求

- 模块文档面、项目级盘点、官方入口事实与当前实现保持一致。
- 不新增 Tool8 或 Tool12+，除非后续单独获得入口治理任务授权。
- 稳定工具能力若被 T08 吸收，T00 文档应同步降级为历史追溯口径。
