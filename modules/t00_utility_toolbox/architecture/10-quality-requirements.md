# 10 质量要求

## 1. 可理解

- Tool1 / Tool2 / Tool3 的范围与非范围必须能从文档直接读出

## 2. 可重复

- Tool1 / Tool2 / Tool3 都必须允许复跑
- 覆盖口径必须稳定为“旧输出先删除再重建”或 Tool1 的“只清空目标 `Vector/` 后重拷贝”

## 3. 可诊断

- 单个 Patch 失败必须保留异常原因
- 全流程结束必须给出总量统计和每个 Patch 的处理结果
- Tool2 / Tool3 的缺失输入与异常 Patch 必须可追溯

## 4. 可治理

- 文档、契约与实现之间边界清晰
- 新工具进入 T00 前必须先补文档
- 不允许借 T00 绕过业务模块治理
