# Contract: 外部字段名解析

## 输入

- `properties`: 外部文件单个要素/记录的属性映射。
- `candidates`: 模块契约已声明的一个或多个逻辑字段名，按优先级排序。

## 行为

1. 每个候选名与实际字段名均通过 `str(...).casefold()` 比较。
2. 第一个存在的 candidate 获胜；不同 candidate 之间不做值合并。
3. 同一 candidate 命中多个仅大小写不同的原字段时：
   - 不同非空值：抛 `FieldNameConflictError`；
   - 相同值或仅一个非空值：返回唯一有效值；
   - 全为空：返回 `None`。
4. `get` 在无匹配时返回调用方给定 default；`require` 在无匹配时抛出缺字段错误。
5. 原始属性映射不得被修改。

## 非目标

- 不推断字段业务含义。
- 不对字段值进行归一化。
- 不自动改写输出字段名。
- 不修复 CRS、几何或拓扑。
