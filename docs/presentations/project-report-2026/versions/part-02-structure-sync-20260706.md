五、第 5 页内容块 1：SWSD Road / Node 字段问题

模块标题：
01 SWSD Road / Node 字段问题

业务意义：
增强信息含量，提供后续模块可用的信息内容。

需要表达的点：

1. Road 的 PatchID
- Road 的 PatchID 可与 RCSD、Patch 内 RC 数据配套。
- 理论上可大幅提高后续业务质量。
- 当前由于 RCSD 与 PatchID 不同源，暂未深度使用。
- PPT 中应表述为“已补齐并保留为后续高价值字段，目前尚未作为主规则深度使用”。

2. Road 的 kind
- Road 的 kind 是原始 SWSD 的道路属性，具有重要参考价值。
- 理论上可大幅提升后续业务质量。
- 当前仅用于少量异常处理，暂未深度使用。
- PPT 中应表述为“保留原始 SWSD 道路属性价值，为后续规则扩展预留空间”。

3. Node 的 kind / grade → kind_2 / grade_2
- Node 原始 kind / grade 不能直接作为后续强规则字段。
- T08 将其初始化为后续稳定消费的 kind_2 / grade_2。
- 下游模块以 kind_2 / grade_2 作为结构判断基础，避免直接依赖不稳定原始字段。

4. 环岛识别与 mainnode 聚合
- Node kind 增加环岛表达。
- 对环岛进行 mainnode 聚合，简化后续路口锚定和 Segment 处理。
- 避免下游把环岛内部多个节点当成普通独立路口处理。

建议视觉表达：
左侧写“业务意义”，中间写“T08 处理”，右侧写“下游价值”。

示例文案：
业务意义：
- 增强 Road / Node 信息含量。
- PatchID 与 kind 保留后续高价值字段。
- kind_2 / grade_2 为后续结构判断提供稳定字段。
- 环岛 mainnode 聚合降低下游结构复杂度。

T08 处理：
- 为 Road 补齐 patch_id / kind。
- 将 Node kind / grade 初始化为 kind_2 / grade_2。
- 聚合环岛 mainnode。
- 显性表达事件 Road、环岛等特殊结构。

下游价值：
- 为 T01 提供稳定 Segment 构建字段。
- 为 T03 / T04 提供统一路口类型输入。
- 为 T05 / T06 提供一致 Road / Node 语义基础。
- 为后续 Patch / RCSD 关系利用预留字段资产。

六、第 5 页内容块 2：路口类型与复杂结构修复

模块标题：
02 路口类型与复杂结构修复

业务意义：
修复 SWSD 数据质量错误，降低后续模块无意义的质量问题。

需要表达的点：

1. T 型路口修复
- Node 中 T 型路口的错误会显著影响路口锚定质量。
- 错误 T 型路口可能导致 T03/T04 误选场景，或导致 T05/T06 的 relation 与 Segment 判断出现无意义失败。
- T08 基于入度 / 出度等结构关系修复明显错误的 T 型路口。

2. 去除无意义交叉路口类型
- 部分 Node 被表达为交叉路口，但实际不应作为路口锚定对象。
- 典型如 SC 变 DC 等场景，会导致非必要的路口锚定要求。
- 去除无意义交叉路口类型，可以减少非必要的锚定约束，提高后续路段替换能力。

3. 分歧 / 合流 / 连续分合流修复
- 连续分歧、合流可能在原始 SWSD Node 中表达不稳定。
- T08 可识别并修复部分错误分歧 / 合流类型。
- 该类修复降低后续 T04 复杂场景处理负担。

4. 复杂路口 kind 增强
- Node kind 增加复杂路口表达，尤其是连续分歧 / 合流等复杂场景。
- 复杂路口作为专项类型进入后续处理策略。
- 在复杂路口内，SWSD 和 RCSD 的挂接关系可能完全不同，必须先在 SWSD 结构输入层面显性化，否则后续模块会承担过多反推压力。

5. RCSDIntersection 一对多预处理
- 原始路口面可能出现一个 RCSDIntersection 对应多个 SWSD 语义路口的情况。
- 这会影响后续路口锚定和 relation 发布。
- T08 可做预处理，减少 T05/T06 阶段的无意义冲突。

6. 持续质量修复策略
- 当前还在持续发现类似 Node kind 错误。
- 后续会追加质量修复策略。
- T08 具备“规则修复 + 人工确认”的双轨机制。
- Tool6 类能力用于输出人工质检候选，不直接绕过人工确认做 silent fix。
- 人工确认后的修复经验可沉淀为后续稳定规则。

建议视觉表达：
业务意义：
- 修复 SWSD 基础结构错误。
- 降低后续无意义锚定失败。
- 减少非必要路口锚定要求。
- 为复杂路口提供专项输入类型。

T08 处理：
- 基于入度 / 出度修复错误路口类型。
- 对提前右转、辅路等场景做豁免复算。
- 聚合连续分歧 / 合流，补充复杂路口表达。
- 处理错误一对多路口关系。
- 追加 Node kind 错误修复策略，并保留人工介入标准能力。

下游价值：
- T03 更准确聚焦交叉 / T 型虚拟锚定。
- T04 更准确聚焦分歧 / 合流 / complex 场景。
- T05 接收更稳定的路口语义并统一发布 relation。
- T06 减少因错误路口类型导致的无意义替换损失。
- 提高路段替换能力。
