# 待确认问题

本文件记录后续继续讨论 PPT 时需要确认的问题。

## 汇报定位

1. 完整母版是否以“项目技术汇报”为主标题，还是使用更业务化的标题。
2. 是否需要单独准备领导精简版，还是先只维护完整母版。
3. 是否需要英文术语对照页，例如 `Movement`、`junction`、`replacement plan`、`problem registry`。

## 项目总览

1. 第 1 页一句话结论的最终表述需要确认。
2. 第 2 页 dashboard 需要哪些关键指标。
3. 第 3 页业务流程图是否按“业务流”表达，还是同时标注 `T03/T04/T05/T06/T10`。
4. 第 4 页演进趋势中，局部更新、高阶要素、Skill 反馈三者的优先级需要确认。

## 成果现状与根因分析

1. 成果漏斗的分母和阶段口径需要确认。
2. 哪个 run root 作为本次汇报的正式结果来源需要确认。
3. 典型 case 选择标准需要确认：代表性优先，还是视觉表达清晰优先。
4. 损失原因是否需要区分“业务合理保留”和“待修复失败”。
5. 示例图片是否从真实 GIS 截图生成，还是允许部分示意图。

## 技术方案

1. 技术方案是否按业务链路组织，而不是按模块编号组织。
2. 每个模块需要展开到什么深度。
3. 是否需要为每个模块设置统一小节：业务目标、业务逻辑、关键技术、质量问题。
4. 是否需要把具体字段和产物路径全部放入附录。

## 道路架构和大模型演进

1. 道路结构层是否需要单独定义 `路口 / 路段 / Movement`。
2. 高阶要素部分是否只讲方向，还是列出具体可落地要素。
3. Skill 反馈机制是否需要展示真实问题闭环案例。
4. 是否需要明确下一阶段路线图和里程碑。
