# 仓库级执行规则

本文件只放**会话级硬约束**。软建议、流程长描述、会过期的事实清单、环境信息表，一律不进入本文件。
若本文件条款之间冲突，由 §1 裁定。

## §1 硬停机触发（任一命中必须立刻停机并回报）

发生以下任一情况，**禁止继续写代码或文档**，必须停下并向用户回报：

1. 项目级源事实之间、模块级源事实之间，或源事实与任务书之间存在冲突。
2. 当前任务书未授权，但本轮改动会触及：
   - 任务书显式排除的模块、目录或 spec；
   - 模块官方对外接口（`INTERFACE_CONTRACT.md`、CLI 子命令、入口脚本签名）；
   - 项目级源事实文档（`SPEC.md`、`docs/PROJECT_BRIEF.md`、`docs/architecture/*`、`docs/doc-governance/module-lifecycle.md`）。
3. 当前任务书未授权，但本轮会**新增长期保留的正式执行入口**：`Makefile` 常驻目标、`scripts/` 与 `tools/` 下常驻命令、模块内 `__main__.py` / `run.py`、CLI 子命令，以及任何应登记到 `docs/repository-metadata/entrypoint-registry.md` 的入口。一次性实验脚本不在本条触发范围内。
4. 命中 §3 中任何一条文件体量禁止行为。
5. 数据现象与已确认的字段语义冲突，准备据此反推上游字段含义并固化为强规则。
6. 用户提供的路径与当前会话 shell 环境不一致（例如用户给的是 WSL 路径但当前会话是 PowerShell），未先做换算与确认。
7. 本轮属于**入口变更任务**（新增 / 删除 / 重命名 / 改变官方调用方式），但 `entrypoint-registry.md` 与 `src/rcsd_topo_poc/cli.py`、`scripts/` 或模块契约的事实不一致。

回报内容必须包括：触发的条款编号、当前事实、缺失/冲突点、建议的下一步选项。

## §2 阅读链路与源事实

- 主阅读入口只有一处：`docs/doc-governance/README.md`。本文件不再维护第二份阅读顺序。
- 项目级源事实集合：`SPEC.md`、`docs/PROJECT_BRIEF.md`、`docs/architecture/*`、`docs/doc-governance/module-lifecycle.md`。
- 模块级源事实集合：`modules/<module>/architecture/*` 与 `modules/<module>/INTERFACE_CONTRACT.md`。
- `specs/*` 是 spec-kit 变更工件，不是源事实；任务未显式要求时不当作 day-0 主阅读材料。
- 默认主阅读 / 主搜索路径**不包含**：`outputs/`、`outputs/_work/`、`.claude/worktrees/`、`.venv/`、`.idea/`、临时审计工件。仅在任务明确要求时进入。

## §3 文件体量硬约束（对应 §1.4，全文最强约束）

- **硬阈值**：单个 `.py` / `.sh` / `.cmd` / `.ps1` / `.ts` / `.js` / `.bat` 等源码 / 脚本文件，**100 KB**。`tests/`、`tools/`、`scripts/` 同样适用。新文件、迁移后文件、重命名后文件一并适用，不存在豁免对象。
- **前置自检（必须，无例外）**：在对任何源码 / 脚本文件**写入或追加**之前，必须先确认该文件**当前**字节数。可用 `Get-ChildItem`（PowerShell）、`stat -c%s`（POSIX）或等效手段。无前置自检即写入，视为违规。
- **三类禁止行为（命中即按 §1.4 停机）**：
  1. 向当前体量已 ≥ 100 KB 的文件追加任何内容（包括注释、空行、import）。
  2. 让某个原本 < 100 KB 的文件因本轮新增首次跨过 100 KB。
  3. 拆分、迁移或重命名已超阈值文件，未在**同一轮**更新 `docs/repository-metadata/code-size-audit.md`。
- **唯一可继续推进路径**（必须在停机回报后由用户授权）：
  - 提交"拆分计划"：列出目标文件、拆分后子模块、契约迁移、测试迁移、风险点。
  - 或提交"豁免说明"：说明本轮无法拆分的原因、新增内容的最小化策略、何时启动拆分；豁免说明必须**同轮**更新 `code-size-audit.md`。
- **登记同步**：任何让 `code-size-audit.md` 表内容应改变的轮次，必须同轮修改该表，否则视为治理缺口、本轮不允许 close。
- **强引用**：凡涉及体量、入口或依赖治理的改动，**必须先读** `docs/repository-metadata/code-boundaries-and-entrypoints.md`。

## §4 范围与变更边界

- 无明确任务书时，不创建具体业务模块实现，不修改未来模块的接口假设，不把骨架治理顺手扩大为业务算法开发。
- 任务书显式排除的模块、目录、spec 一律视为保护区；除非用户重新授权，不得顺手触碰。
- 不在 `main` 上直接做中等及以上结构化治理变更；该类变更走 SpecKit。
- 不修改与当前任务无关的代码、注释、文档、命名和结构。
- 不删除未完全理解的注释、历史逻辑或兼容代码。
- 模块根目录**禁止**新增 `SKILL.md`；标准可复用流程统一放 `.agents/skills/<skill-name>/SKILL.md`。
- `modules/_template/` 仅作为模板，不是业务模块，不参与生命周期盘点。

## §5 数据语义与字段管控

- 禁止根据局部样本、人工真值或单次冒烟结果，自行反推上游字段语义并固化为强规则；命中按 §1.5 停机。
- 未在项目 / 模块源事实文档中正式启用的输入字段，**不得**进入 Step1 / Step2 强规则。
- 字段一旦正式启用，**必须**在同一轮把该字段写入项目级约束与对应模块契约，并说明：当前可用语义、适用范围、未确认边界。
- GIS / 拓扑 / 空间数据相关任务，必须显式覆盖以下检查项（缺一项即视为未完成）：
  - CRS 与坐标变换正确性；
  - 拓扑一致性（不允许 silent fix）；
  - 几何语义可解释性；
  - 审计可追溯性（输入、参数、输出、运行环境可定位）；
  - 性能可验证性。

## §6 流程路由

- 默认编程流程以 `.agents/skills/default-imp/SKILL.md` 为唯一真相，本文件不重复其守则。
- 正式大型任务（需求新增 / 跨模块重构 / 影响正式业务口径 / 需要明确 `specify / plan / tasks` 的任务）**必须**走 SpecKit，主流程固定为 `specify / plan / tasks / implement`。
- SpecKit 任务书必须覆盖产品 / 架构 / 研发 / 测试 / QA 五类职责视角；测试与 QA 视角缺失即视为任务书未就绪，不得进入 implement 阶段。
- SpecKit 的 implement 阶段默认遵循 `default-imp/SKILL.md` 的执行纪律。
- 任务执行中如发现边界不清、影响扩大、业务口径不稳，必须从 default-imp 升级为 SpecKit。
- **完成回报最小集（强制）**：每轮交付必须区分 **已修改 / 已验证 / 待确认** 三档，并列出修改的文件路径与每个文件的目的。"看起来应该可以"不得表述为"已经修复"。

## §7 执行环境与边界

- 本文件不预设默认 shell；当前会话的 shell 以系统注入信息为准。涉及跨盘 / 跨环境路径时，按 §1.6 处理。
- 跨盘符路径换算与 `TestData/POC_Data` 根目录约定见 `docs/repository-metadata/path-conventions.md`；该文件未建立前，沿用历史 `AGENTS.md` 中的盘符与测试数据根目录约定，并把"建立 path-conventions.md"列为治理缺口。
- Agent 可执行范围：外网验证、外网数据检查、本地工作区操作。
- 内网环境、内网数据拉取、内网命令默认不可由 Agent 执行；除非用户在当轮明确提供了可执行的内网访问能力。
- 不得把内网操作误表述为自己已实际执行。

## §8 文档语言

- 仓库内文档默认中文；参数、代码、命令、路径、模块标识、配置键、接口字段可保留英文。
- 本文件本身使用中文；不在常规变更轮次中改写为英文。
