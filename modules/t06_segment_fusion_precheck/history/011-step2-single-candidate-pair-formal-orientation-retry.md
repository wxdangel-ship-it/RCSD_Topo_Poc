# 011 Step2 单向候选 pair 正式方向试算重试

- 时间：2026-06-12
- 背景：T10 剩余 `required_semantic_nodes_not_connected_in_buffer` 漏斗中，部分高等级单向 SWSD Segment 的 50m buffer 内存在可用 RCSD corridor，但 SWSD/RCSD 语义路口锚定差异导致原始 RCSD pair 不适合作为当前 Segment 的正式端点；同时单向分歧合流场景下，`BufferOnlyProbe` 按 SWSD 端点距离给出的候选 pair 方向可能与 RCSD 有向通行方向相反。
- 根因：既有 `pair_anchor_auto_retry.py` 只构造一个 effective relation，并先走常规 50m buffer extractor；当候选 pair 方向与单向 RCSD 有向边相反时，正式抽取会因 `rcsd_directed_path_missing` 失败，后续 adaptive / graph-first retry 又回到原始 relation，无法组合“pair anchor 修正 + 单向纵向 graph-first 校验”。
- 变更：新增 `pair_anchor_formal_retry.py`，仅对原始失败原因为 `required_semantic_nodes_not_connected_in_buffer`、非人工复核的 `high_confidence_pair_anchor_candidate`、`failure_business_category=pair_anchor_mismatch`、`directionality=single` 场景生效；只取 `BufferOnlyProbe.candidate_pair_sets[0]`，分别按 as-is 与 reversed 两个方向构造临时 T06 relation，先跑正式 `BufferSegmentExtractor`，失败后再跑既有 `SingleGraphConnectivityRetry`。只有恰好一个方向通过正式硬审计时才返回 replaceable outcome。
- 输出：新增 `pair_anchor_formal_retry_rows.py`，将通过 outcome 统一写入 `buffer_only_probe`、`repair_candidates`、`candidates`、`replaceable` 与 `failure_business_audit`；若 graph-first 被使用，继续记录 `adaptive_buffer_status`、`adaptive_buffer_distance_m` 与 `adaptive_buffer_source_reason`。
- 安全边界：不回写 T05 relation，不修改输入字段语义，不读取未授权属性，不扩大整体 buffer；常规原始 relation 与已授权 adaptive retry 优先，formal orientation retry 只作为原始 relation 失败后的补漏兜底，不能覆盖已经可替换的 Segment 或已经通过 adaptive 的结果；低排名候选 pair 即使在离线重算中可通过，也不进入自动替换，避免在多候选通过时做隐式业务选择。最终成段必须经过正式 buffer / direction / topology / geometry hard audit，或经过 single graph-first 的 50m core overlap、端部外延、长度比与几何覆盖门槛。
- GIS / 拓扑检查：CRS 沿用 Step2 输入归一化后的 `EPSG:3857`；候选 RCSD pair 必须在 RCSD graph 中满足单向 directed path；几何语义仍以 SWSD Segment 定义 50m core buffer，RCSD 只允许通过该 core 后按纵向联通补足；审计可追溯到原始 pair、候选 pair、方向、reject reason、adaptive source 与输出文件；性能边界为每个命中 Segment 最多试算排名第一候选 pair 的两个方向。
- 回归：新增 `test_step2_formal_retry_reverses_single_candidate_pair_when_direction_requires`，覆盖候选 pair as-is 方向缺失有向 path、reversed 方向通过正式 extractor 并进入 replaceable 的场景；同步复跑 `test_step2_retries_high_confidence_candidate_anchor_mismatch` 与 `test_step2_high_grade_single_retries_graph_first_without_anchor_change`，确认既有 pair-anchor retry 与原始 relation adaptive retry 仍保持优先。
