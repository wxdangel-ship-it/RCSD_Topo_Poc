# 11 风险与技术债

## 当前主要风险

1. 当前输入数据组织方式只是兼容参考仓库，未来可能与 RCSD 真实业务需求分叉。
2. 项目级文档、模块登记文档、入口注册表和真实实现入口容易发生漂移。
3. 共享协议当前是仓库级通用版本，后续某些模块可能需要扩展指标与断点枚举。

## 当前可接受技术债

- 当前存在超阈值源码 / 测试文件，后续触碰时必须补拆分计划或豁免说明。
- 当前历史工件与临时工件边界需要持续维护，避免侵入主阅读路径。

## 后续缓解方式

- 任何 RCSD 模块启动任务都先基于模板补齐文档契约
- 数据布局调整必须通过独立任务写回项目级文档
- 共享协议扩展时同步补测试
- 项目级盘点、生命周期与入口注册表必须与真实仓库状态同步
