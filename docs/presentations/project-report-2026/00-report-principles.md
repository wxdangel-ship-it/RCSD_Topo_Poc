# 汇报原则

## 汇报目标

本次汇报先形成完整母版，再从母版裁剪面向不同受众的版本。完整母版需要同时覆盖：

- 项目为什么有业务价值；
- 当前实际做到了什么；
- 损失和质量问题是什么原因造成的；
- 技术方案如何支撑这些结果；
- 后续如何从一次性全量融合演进为持续优化能力。

## 受众策略

大多数听众不关心理论，也不熟悉架构。因此汇报顺序采用：

1. 先讲效果，再讲技术。
2. 先讲业务收益，再讲架构抽象。
3. 先讲典型案例，再讲模块实现。
4. 先讲当前结果和损失，再讲道路结构层、Movement 和 Skill 机制。

## 表达原则

- 不从理论开场，以项目效果和核心结论开场。
- 不把 SWSD 和 RCSD 讲成对等建模来源。道路结构层以 SWSD 为核心构建，RCSD 通过真实证据锚定后作为融合承载网络参与替换。
- 不把技术方案讲成模块流水账，每个模块都必须回答“它如何让最终结果更可靠”。
- 不把 PPT 里的数字写成孤立结论，每个数字都要能追溯到证据。
- 不把质量问题包装成已解决，必须区分已闭环、已定位待验证、待业务确认和长尾数据问题。

## 核心叙事

本项目不是一次简单的数据拼接或单点算法修复，而是以 SWSD 为核心构建路口、路段、Movement 道路结构认知层，再用真实证据将 RCSD 锚定到这套结构上，形成可解释、可审计、可持续优化的全量融合链路。融合的本质是在保留全量拓扑通行能力的前提下，将承载骨架替换为 RCSD 的高精度骨架路网。RCSD 的工艺标准和质量现状不足以独立支撑道路结构层构建，因此项目不以 RCSD 直接反推核心结构。

当前技术形态以一次性全量融合为主，用于形成统一 baseline 和质量漏斗；后续可演进为按区域、路口、道路变更范围或质量问题反馈触发的局部更新能力。

## 术语表达

| 技术表述 | 汇报表达 |
|---|---|
| 道路结构层 | 以 SWSD 为核心建立路口、路段和通行结构认知，再牵引 RCSD 融合判断。 |
| Movement | 路段-路口-路段的通行关系单元。 |
| 异源数据融合 | 以 SWSD 道路结构为牵引，通过路口面、道路面、导流带等真实证据建立 SWSD-RCSD 锚定，在保留全量拓扑通行能力的前提下构建 RCSD Segment 并替换为 RCSD 高精度骨架路网。 |
| 一次性全量融合 | 当前阶段先完成全量治理、统一 baseline 和质量漏斗。 |
| 局部更新能力 | 后续可按区域、路口或变更影响范围做增量重算。 |
| 大模型 Skill 反馈 | 把反复出现的质量问题、业务判断和修复经验沉淀为长期优化机制。 |
