# 完整母版目录

本文件记录完整 PPT 母版的章节结构。精简版应从本目录裁剪，不重新设计主线。

## 1. 项目总览

本章固定 4 页，是整场汇报最重要的定调部分。目标是让听众快速理解项目价值、关键成果、整体流程和演进方向。

### 1.1 一句话结论与核心价值

表达项目的总判断：本项目以 SWSD 道路结构认知为牵引，利用真实证据建立 SWSD-RCSD 路口锚定，在保留全量拓扑通行能力的前提下构建 RCSD Segment、替换为 RCSD 高精度骨架路网，并建立可审计、可追溯、可持续优化的质量闭环。

### 1.2 关键成果摘要

用 dashboard 方式呈现处理范围、关键成功结果、主要质量提升、当前交付能力和剩余问题分类。

### 1.3 总体业务流程

用业务流说明项目不是直接拼接数据，也不是用 RCSD 反推核心结构，而是先以 SWSD 构建路口、路段、Movement，再基于真实证据建立 SWSD-RCSD 路口锚定，最后以 SWSD Segment 和锚定路口牵引 RCSD Segment 构建，在保留全量拓扑通行能力的前提下完成高精度骨架路网替换、结果输出和质量审计。

### 1.4 项目演进趋势

说明当前阶段是一次性全量融合，后续可演进为局部更新、高阶要素生成和 Skill 化质量反馈。

## 2. 成果现状与根因分析

本章以关键结果为脉络展开漏斗式分析，先讲结果，再讲损失原因。

建议漏斗主线：

1. 总输入规模。
2. 可进入结构识别的数据规模。
3. 成功构建 SWSD 路口、路段、Movement 的规模。
4. 基于真实证据成功建立 SWSD-RCSD 路口锚定的规模。
5. 在 SWSD Segment 和锚定路口牵引下可构建 RCSD Segment、可融合或可替换的规模。
6. 最终成功输出的规模。
7. 损失、保留、失败、待确认部分。

损失原因建议分组：

- 数据缺失。
- 几何差异。
- 路口复杂。
- 关系冲突。
- 替换风险。
- 工程运行和审计一致性问题。

典型示例采用固定表达：

`现象 -> 根因 -> 当前处理策略`

## 3. 技术方案

本章按模块展开，但不做模块流水账。每个模块统一回答：

- 模块业务目标是什么；
- 它解决哪类核心问题；
- 输入和输出是什么；
- 主要业务逻辑是什么；
- 关键技术点是什么；
- 当前质量问题是什么；
- 它对最终成果漏斗的贡献是什么。

建议按业务链路组织：

1. 道路结构准备与基础清洗。
2. 路口、代表节点、虚拟 junction 构建。
3. SWSD/RCSD 关系构建。
4. 融合与替换资格判断。
5. 替换执行与结果生成。
6. 全链路编排、报告与质量审计。

## 4. 道路架构和基于大模型的演进

本章负责说明项目的长期价值，但仍然以效果语言表达。

### 4.1 道路结构层架构

说明为什么以 SWSD 构建的路口、路段、Movement 可以作为融合、更新和高阶要素构建的底座，以及为什么 RCSD 需要通过真实证据锚定后参与承载替换。

### 4.2 从全量融合到局部更新

说明当前一次性全量融合如何演进为按区域、路口、道路变更范围或质量问题反馈触发的局部更新。

### 4.3 高阶要素自动构建

说明道路结构层稳定后，如何继续支撑转向、通行组织、禁限行、车道级关系、路网连通性等高阶要素。

### 4.4 大模型 Skill 反馈机制

说明如何把质量问题、根因分析、业务判断和修复经验沉淀为长期优化机制。
