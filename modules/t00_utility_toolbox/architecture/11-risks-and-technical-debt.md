# 11 风险与技术债

## 当前主要风险

- Tool1 可能被顺手扩展为点云、轨迹或复杂治理流水线
- Tool2 / Tool3 可能因未来增量需求被提前抽象成重型框架
- Windows 路径与 WSL 路径若映射不清，容易导致后续实现时读写目标错误
- 当前环境缺失默认 `patch_all` 真实数据根，真实全量验证会受数据可达性影响

## 当前可接受技术债

- `history/*` 暂未建立，待进入真实迭代后再补
- 当前仍不为 Tool4+ 预建设计
- GeoJSON 缺失 CRS 时仍需依赖脚本头部的默认 CRS 配置

## 后续缓解方式

- 继续以 `spec.md` 与 `INTERFACE_CONTRACT.md` 约束 Tool2 / Tool3 的最小实现边界
- 当真实数据路径可达后，再补足全量运行验证结果
- 若新增工具，逐个补规格并单独评审
