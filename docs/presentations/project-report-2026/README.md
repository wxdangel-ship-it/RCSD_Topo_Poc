# 项目技术汇报工作包

本目录用于沉淀 `RCSD_Topo_Poc` 项目技术汇报的结构、内容草稿、素材索引和最终交付物。

## 定位

- 本目录是汇报交付工作包，不是正式业务模块。
- 本目录不替代项目级源事实、模块级源事实、接口契约或运行审计工件。
- PPT 中的结论、数字、案例和图示，必须能回到项目文档、模块文档、真实 run root、审计 CSV/JSON 或人工确认记录。
- 精简版汇报应从完整母版中裁剪，不单独重写一套叙事。

## 当前确认的汇报结构

1. 项目总览
2. 成果现状与根因分析
3. 技术方案
4. 道路架构和基于大模型的演进

其中“项目总览”固定控制为 4 页：

1. 一句话结论与核心价值
2. 关键成果摘要
3. 总体业务流程
4. 项目演进趋势

## 目录说明

| 路径 | 用途 |
|---|---|
| `00-report-principles.md` | 汇报原则、受众策略和表达边界。 |
| `01-master-outline.md` | 完整母版章节结构和叙事顺序。 |
| `02-slide-catalog.md` | 每页 PPT 的标题、结论、证据、图片需求和裁剪策略。 |
| `03-evidence-index.md` | 汇报结论对应的证据索引，避免无来源数字和判断。 |
| `04-metric-system.md` | 汇报指标体系，定义端到端、路口锚定、Segment 替换和 restriction 恢复的指标口径。 |
| `05-open-questions.md` | 后续讨论中需要确认的问题。 |
| `slides/` | 分章节 Markdown 草稿。 |
| `assets/diagrams/` | 汇报用流程图、架构图、漏斗图等轻量图示。 |
| `assets/screenshots/` | 汇报用界面截图、运行结果截图。 |
| `assets/case-images/` | 典型 case 图片、前后对照图和问题示意图。 |
| `deck/` | 最终 PPT/PDF 结果。 |
| `versions/` | 阶段性讨论稿、内容快照和版本说明。 |

## 使用规则

- 先维护 `02-slide-catalog.md`，再展开单页内容。
- 每个关键指标先登记到 `03-evidence-index.md`，再进入 PPT。
- 示例图片必须标注来源 case、run root 或人工绘制状态。
- 面向领导的精简版只从 `02-slide-catalog.md` 中筛选，不改变母版主线。
- 技术细节进入正文还是附录，由“是否支撑核心效果判断”决定。
