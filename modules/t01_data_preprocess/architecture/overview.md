# T01 高层概览

## 1. 已确认事实：T01 在 RCSD 项目中的位置

- T01 是 RCSD_Topo_Poc 中的数据预处理模块候选文档面。
- 当前仓库仍处于“先建治理底座，再建业务模块”的阶段。
- 因此，T01 当前只完成文档框架初始化，不代表模块已经进入实现交付阶段。
- 当前版本也不改变项目级生命周期盘点中“尚无正式业务模块”的口径。

## 2. 当前理解/归纳：当前优先子能力在整体流程中的作用

- 路段提取：用于把交叉路口之间的道路段落问题先转化为可讨论的预处理对象。
- 路口类型重新赋值：用于支撑后续对 T 型、分歧、合流等路口语义的整理与确认。
- 当前理解下，这两个子能力都属于“上游数据整理与语义澄清”范围，而不是最终业务成果本身。

## 3. 已确认事实：当前上游输入假设

- 上游提供两类图层：
  - `Road` 线图层
  - `Node` 点图层
- 图层路径与图层名由参数传入。
- `Road` 当前已知关键关联字段为 `snodeid`、`enodeid`。
- `Node` 当前已知关键聚合字段为 `mainnodeid`，且其是当前语义路口聚合主依据。
- 输入在进入 T01 前 / 中会统一做 CRS 归一化，目标 CRS 为 `3857`。
- 输入字段会统一做归一化处理。
- 当前输入异常处理口径为报异常，但不中断其它对象处理。
