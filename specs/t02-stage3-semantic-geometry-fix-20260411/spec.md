# T02 Stage3 语义成功与几何形态修复规格

## 1. 文档定位

- 类型：定向修复规格
- 范围：`t02_junction_anchor` 的 Stage3 `virtual_intersection_poc`
- 目标：在不破坏现有 Stage3 测试样例集的前提下，修复 `706389`、`761318` 的业务语义成功，并对最终路口面几何做一轮认知一致性优化

## 2. 本轮业务目标

### 2.1 语义成功目标

- `706389` 必须从当前“语义缺口导致 review_required”收敛为业务意义上的成功
- `761318` 必须从当前“T 型口 RC 语义不完整导致 review_required”收敛为业务意义上的成功
- `520394575` 继续作为硬失败样例保留，不得被放行
- 修复后不得让现有 Stage3 case-package 样例集中原本稳定成功的 case 退化为失败

### 2.2 几何形态目标

- 保持业务意义成功的同时，最终 polygon 应更接近人工认知中的单一连续路口面
- 不得出现无意义的狭长连接
- 不得出现无意义的空洞
- 不得出现无意义的凹陷
- 不得为了平滑几何而重新吞入其他 `SWSD` 语义路口或其外延道路

## 3. 冻结约束

- `SWSD` 语义路口仍是 Stage3 的唯一基础语义
- `RCSD` 只用于补强、补齐、校验，不得反客为主替代 `SWSD`
- 任何修复不得把 `520394575` 从硬失败改成成功
- 任何几何优化不得放宽对 foreign `SWSD node/road` 的约束

## 4. 第一阶段验收

- `706389`：必须进入 `success=true`，且关键 `RCSDNode` / `RC group` 语义完整
- `761318`：必须进入 `success=true`，且 T 型口对应 `RCSDNode` 被正确纳入；若仍无法正确补齐，则至少不得误放为成功
- `520394575`：仍为 `success=false`，且失败渲染样式保持契约要求

## 5. 第二阶段验收

- 在第一阶段通过基础上，对 Stage3 全量本地 case-package 结果做几何收口
- 重点关注：
  - `707267`
  - `758888`
  - 与本轮新增修复直接相关的 `706389`、`761318`
- 几何优化后必须再跑随机 10 例护栏和全量本地 visual 包

## 6. 回归护栏

- Stage3 单测必须保持全绿
- 随机抽取 10 个现有用例做护栏回归
- 最终必须输出全量本地 case-package visual 包供人工目视检查
